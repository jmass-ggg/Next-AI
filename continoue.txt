5. Fixed cookie-based authentication

You learned that this code:

expects this header:

Cookie: access_token=YOUR_TOKEN

Not this:

Authorization: Bearer YOUR_TOKEN

Then we added support for both cookie and Authorization header.

6. Added Bearer token authentication

We added:

def get_bearer_token(self):
    auth_header = self.req.headers.get("Authorization")

    if not auth_header:
        return None

    if not auth_header.startswith("Bearer "):
        return None

    return auth_header.split(" ", 1)[1]

Now your API can accept:

Authorization: Bearer ACCESS_TOKEN

This is useful for protected APIs.

7. Built reusable auth checker

We created this idea:

def require_auth(self):
    token = self.get_bearer_token()

    if not token:
        token = self.get_cookie("access_token")

    if not token:
        self.response({"error": "Login required"}, status=401)
        return None

    email = verify_access_token(token)

    if not email:
        self.response({"error": "Invalid or expired access token"}, status=401)
        return None

    return email

This makes protected routes easier.

Later we improved the idea to:

def require_user_id(self):
    ...
    return user_id

So every protected API can know which user is logged in.

Final backend pattern we reached

For any protected API:

def some_protected_api(self):
    user_id = self.require_user_id()

    if not user_id:
        return

    data = self.read()

    self.response(
        self.service.some_action(user_id, data)
    )

This is the main engineering idea:

Access token proves who the user is.
Backend extracts user_id from token.
Backend uses that user_id for database queries.
User cannot fake another user's id from request body.

That is real authentication + authorization flow.
